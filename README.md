**Назначение**

Описание: этот репозиторий содержит релизы и инструкции для установки Ryazhenka CFW. Материалы предназначены для опытных пользователей, знакомых с процедурами загрузки кастомной прошивки на Nintendo Switch.

---

**Ключевые компоненты (обязательное выделение)**

- `Ryazhahand-Overlay` — система оверлеев для управления контроллерами и пользовательских настроек геймпада. Краткое назначение: кастомизация назначений кнопок, управление профилями и доступ к быстрым настройкам из оверлея. Разработано: Dimasick-git.
- `Ryazha-clk` — модуль управления частотами CPU/GPU и профилями производительности. Краткое назначение: применение и переключение профилей частот, сохранение пользовательских конфигураций, безопасное изменение параметров управления питанием. Разработано: Dimasick-git.
- `Ryazhenka Tuner` — инструмент тонкой настройки прошивки и системных параметров. Краткое назначение: конфигурирование системных параметров, скрипты автооптимизации и инструменты диагностики. Разработано: Dimasick-git.

Примечание: перечисленные модули являются основными компонентами экосистемы Ryazhenka и должны рассматриваться как первоочередные при настройке системы.

**Зависимости и совместимость**

- Bootloader: Hekate (версия совместима с релизом). Рекомендуется использовать официальные сборки CTCaer.
- Custom Firmware: Atmosphere (совместимые версии). Требуется совместимая версия Atmosphere для корректной работы модулей.
- Sigpatches: требуются для запуска неподписанных компонентов; поставляются отдельно и не включены в этот репозиторий.

---

**Установка (кратко, технически)**

1. Скачайте релиз из раздела Releases этого репозитория.
2. Распакуйте архив на корень SD-карты (FAT32/exFAT в зависимости от конфигурации).
3. Убедитесь, что в SD находятся необходимые файлы Hekate/Atmosphere согласно документации используемой версии bootloader/CFW.
4. Запустите устройство в RCM и запустите Hekate, выберите вариант загрузки с SD, соответствующий Ryazhenka.
5. После начальной загрузки проверьте работоспособность основных модулей: `Ryazhahand-Overlay`, `Ryazha-clk`, `Ryazhenka Tuner`.

Примечание: детальные инструкции по установке и восстановлению находятся в `INSTALL.md`.

---

**Конфигурация**

- Файлы конфигурации модулей расположены в каталоге `config/` релиза или в корне SD-карты согласно структуре конкретного релиза.
- Для `Ryazha-clk` конфигурации профилей сохраняются в формате JSON/YAML (см. документацию модуля). Изменения требуют перезапуска службы модуля или перезагрузки системы.
- Для `Ryazhahand-Overlay` пользовательские профили контроллеров хранятся в отдельном каталоге профилей; переключение выполняется через оверлей или CLI-инструмент при наличии.

**Обновление и релизы**

Рекомендации по обновлению:
- Резервная копия текущей SD-карты перед установкой нового релиза.
- Проверка сумм и цифровых подписей релиза (при наличии).
- Чтение файла `CHANGELOG.md` для выявления несовместимых изменений.

Формат релизов (шаблон):
- Заголовок: `vX.Y.Z — краткое описание изменений`
- Разделы: "Изменения", "Исправления", "Известные проблемы", "Инструкции по обновлению"

Пример заметок релиза (технический шаблон):

Changes
- Обновлён `Ryazha-clk`: исправлена логика применения профилей при переходе в спящий режим. (Разработано: Dimasick-git)
- Добавлен режим диагностики в `Ryazhenka Tuner` для сбора логов производительности. (Разработано: Dimasick-git)

Fixes
- Исправлено состояние оверлея при переключении профилей контроллера в `Ryazhahand-Overlay`.

Known issues
- Описание известных ограничений с указанием условий воспроизведения.

Upgrade instructions
- Пошагово: резервная копия → распаковка → проверка прав доступа → загрузка.

**Формат CHANGELOG**

- Разделы по версии в порядке убывания (новые версии сверху).
- Для каждой версии указывать: краткое резюме, подробный список изменений (файлы/модули), автор изменений.

Пример записи:

```
v1.2.0
- Added: Ryazha-clk profile manager (Dimasick-git)
- Fixed: overlay handling in Ryazhahand-Overlay (Dimasick-git)
```
